How we use PhoneGap

- At this point you may be thinking, holy crap, there is more to this gaming stuff that I thought… Well, you can still make awesome offline and simpler games architecturally, however, you'd be right if you want to make successful triple A social games. - Let's focus on PhoneGap for now, hopefully some of the components you see today can be used in your own projects.

PhoneGap as a Framework

    ★ Stable bridging callback API layer (Web <-> native)


    ★ API extensions via plugins


    ★ Device Events: onPause, onResume, etc.


    ★ Cross Platform

- What does PhoneGap do for us? - Provides a stable bridging callback API layer (from Web to native) - Provides API extensions via plugins - we love the browser, but it's limited y'know ;) - Device events: onPause(), onResume(), etc. - (All of the above on every platform!) Cross Platform

Taking Advantage of PhoneGap

(plugins to solve cross platform or web related issues)

- Asset management (storage, steam, download, versioning) - Multiple pages/views - Native Analytics - Spinners / Loaders - SplashScreen (reload the game from session timeout?, show the splash) - Custom components (Ejecta, Platform SDKs) - OS features: - IAP - GameCenter/Google Services - Notifications (local & remote)
